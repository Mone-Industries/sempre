## What This Project Is

sempre is a **Docker-based multi-agent AI team** built on OpenClaw + Ollama.
This is NOT a coding project — work here is editing templates, Docker configs, and agent behavior files.

**Unlimited agents** — defined in `config.json` → `agents[]`
**Containers:** openclaw + ollama + nginx + n8n + (gh-proxy × N + crawl4ai × N + browser-use × N) where N = agent count
**Default team:** 3 agents (mone/GM, sam/worker, nina/worker)

## Architecture — Correct Understanding

### Single Config File: config.json
`config.json` is the **single source of truth** for everything — agents, secrets, ports, models.

- `config.json` → gitignored (contains secrets)
- `config.json.example` → committed to git (template)
- `scripts/generate.sh` reads `config.json` and produces:
  - `.env.generated` (chmod 600) — env vars for docker compose
  - `docker-compose.override.yml` — per-agent services
- `config.json` is mounted directly into the openclaw container at `/opt/sempre/config.json`
- `entrypoint.sh` reads it at runtime — no intermediate `team.json`

### GitHub Access: gh-proxy (NOT github-mcp)
GitHub access is handled by a **custom Go HTTP reverse proxy** at `docker/gh-proxy/`.

- Agent calls `http://gh-proxy-{agent_id}:8080/{any-github-api-path}`
- gh-proxy forwards to `https://api.github.com` with `Authorization: Bearer {GITHUB_TOKEN}` injected server-side
- Agents **never see the token** — fully transparent proxy
- No MCP, no stdio — plain HTTP

> There is NO `github-mcp` container. It was replaced by gh-proxy. Do not add it back.

### Memory & KM: OpenClaw Native (NOT LanceDB or external vector DB)
OpenClaw has **built-in vector search** powered by `nomic-embed-text` via Ollama.

Configured in `openclaw.json` (`memorySearch` section):
```json
{
  "provider": "ollama",
  "model": "nomic-embed-text",
  "sources": ["memory", "sessions"],
  "extraPaths": ["/root/.km"],
  "sync": { "watch": true, "onSessionStart": true }
}
```

- OpenClaw indexes **session memory + past sessions + the entire `/root/.km` Obsidian vault** automatically
- Vault is watched for file changes and re-indexed live
- No external vector DB needed

### KM Vault: Obsidian Format
- Lives at `disks/km/` on host, mounted to `/root/.km/` in container
- Format: Markdown with YAML frontmatter + `[[wikilinks]]` + MOC index files
- **Shared across all agents** — one brain for the whole team
- Drop `.md` files into `disks/km/` → agents find them immediately via semantic search

### Per-Agent Persona
Each agent has a `"persona"` field in `config.json`. This is injected into `SOUL.md` as `${AGENT_PERSONA}` at container startup — each agent has unique personality, not a shared robotic template.

### Model: Single Shared (Default)
All agents use `OLLAMA_MODEL` by default (set via `config.json → llm.model`).
Per-agent model can be set by editing individual agent entries in `openclaw.json.tpl`.

## Common Commands

```bash
make up                  # validate config → cert (auto) → build → start all
make down                # stop all containers
make restart             # stop + start (picks up config.json changes)
make generate            # regenerate .env.generated + docker-compose.override.yml
make logs                # tail all logs
make ssh                 # SSH into openclaw gateway
make shell               # docker exec bash (no SSH)
make status              # container status
```

**After editing `config.json`** (agents, secrets, ports, models):
```bash
make restart
```

**After editing any `.tpl` file** (agent rules/skills — always-overwrite files):
```bash
make restart             # entrypoint re-applies templates on every start
```

**After editing `Dockerfile` or `entrypoint.sh`** (requires image rebuild):
```bash
make rebuild-openclaw
```

## Template Files (`.tpl`) — How They Work

### Categories:

| Category | Files | Behavior |
|----------|-------|----------|
| **Team skills** (shared) | `SKILL.md`, `KM.md`, `WORKFLOW.md`, `COMMUNICATION.md` | Synced to `/root/.openclaw/skills/${TEAM_NAME}/` every restart |
| **Agent workspace** (per-agent) | `SOUL.md`, `AGENTS.md`, `TOOLS.md` | Always-overwrite per agent every restart |
| **First-run only** | `openclaw.json`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md` | Generated once — user can edit freely after |

### Template variables available:

| Variable | Source | Used in |
|----------|--------|---------|
| `${TEAM_NAME}` | `config.json → team` | All skill files |
| `${AGENT_ID}` | per-agent loop (`config.json → agents[]`) | `TOOLS.md`, `IDENTITY.md` |
| `${AGENT_NAME}` | per-agent loop (`config.json → agents[]`) | `SOUL.md`, `IDENTITY.md` |
| `${AGENT_PERSONA}` | per-agent loop (`config.json → agents[].persona`) | `SOUL.md` |
| `${OLLAMA_MODEL}` | `.env.generated` | `openclaw.json.tpl` |
| `${KM_MAX_HOPS}` | `.env.generated` | `KM.md` |

> `AGENTS.md` is generated dynamically in `entrypoint.sh` from `config.json` (not from a `.tpl` file)

## ⚠️ Pattern: Adding a New Skill (3 steps — all required)

### Step 1 — Create template
```
docker/openclaw/<NAME>.md.tpl
```
Frontmatter:
```markdown
---
name: ${TEAM_NAME}-<name>
description: <description> for ${TEAM_NAME} agents
---
```

### Step 2 — COPY into Docker image (`Dockerfile`)
```dockerfile
COPY <NAME>.md.tpl    /opt/openclaw-default/<NAME>.md.tpl
```

### Step 3 — Sync in entrypoint (`entrypoint.sh`)
```bash
TEAM_NAME="${TEAM_NAME}" envsubst '${TEAM_NAME}' \
    < /opt/openclaw-default/<NAME>.md.tpl \
    > "${SKILL_DIR}/<NAME>.md"
```
Update the `echo` line to include `<NAME>` in the list.

## Skills Index

### Team skills (loaded from `/root/.openclaw/skills/${TEAM_NAME}/`)

| File | Purpose |
|------|---------|
| `SKILL.md` | Platform service routing — which tool to use for each task type |
| `KM.md` | Obsidian vault SOP — write and read knowledge |
| `WORKFLOW.md` | Task complexity triage + planning + verification |
| `COMMUNICATION.md` | Structural communication rules — defers personality to each agent's SOUL.md |

### Per-agent workspace (in `~/.openclaw/agents/{id}/workspace/`)

| File | Purpose | Behavior |
|------|---------|----------|
| `SOUL.md` | Identity, persona, tool routing, non-negotiables | Always-overwrite |
| `AGENTS.md` | Roles, delegation rules, workflow pointer | Always-overwrite |
| `TOOLS.md` | Ready-to-use curl commands for this agent's services | Always-overwrite |
| `IDENTITY.md` | Agent ID + name | First-run only |

## Service Topology

```
Discord / Web UI / SSH
         ↓
    nginx (SSL termination — Tailscale cert)
         ↓
    openclaw (gateway + N agents + nomic-embed-text memory)
         ↓
      ollama (LLM — Ollama Cloud or local models)

Per-agent services (N× each — N from config.json agents[]):
  gh-proxy-<id>:8080      ← GitHub REST API proxy (GITHUB_TOKEN pre-injected)
  crawl4ai-<id>:11235     ← Web scraper → clean Markdown
  browser-use-<id>:8080   ← AI browser (click, login, JS, SPA)

Shared services:
  n8n:5678                ← Workflow automation (via nginx SSL)
```

Network: `${TEAM_NAME}-network` (bridge)
All containers communicate by Docker service alias.
